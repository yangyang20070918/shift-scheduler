# 手動テスト計画書

> 本番デプロイ前の手動検証チェックリスト。  
> テスト結果は各項目の `[ ]` を `[x]`（合格）または `[!]`（不合格 — 備考記入）に変更してください。

---

## 前提条件

```bash
# 1. バックエンド起動
cd shift_app
uvicorn api.main:app --host 0.0.0.0 --port 8000 --reload

# 2. フロントエンド起動
cd shift_app/web
npm run dev
# → http://localhost:5173

# 3. 自動テスト通過確認
python -m pytest tests/ -v
# → 62件全通過であること
```

---

## 1. 認証 (Auth)

### 1-1. ユーザー登録
- [ ] http://localhost:5173/login にアクセス → ログイン画面が表示される
- [ ] 「新規登録」タブに切替可能
- [ ] 必要項目（メール、パスワード、名前、組織名）を入力して登録 → 成功
- [ ] 同じメールで再登録 → エラーメッセージが表示される
- [ ] パスワード5文字以下で登録 → バリデーションエラー

### 1-2. ログイン/ログアウト
- [ ] 登録したアカウントでログイン → スケジュール一覧にリダイレクト
- [ ] 画面右上にユーザー名が表示される
- [ ] 「ログアウト」ボタン → ログイン画面に戻る
- [ ] ログアウト後、直接 http://localhost:5173/schedules にアクセス → ログインにリダイレクト

---

## 2. パターン管理 (Patterns)

- [ ] サイドバー「パターン」→ パターン一覧画面
- [ ] 「パターン追加」→ モーダル表示
- [ ] 以下のパターンを作成:
  - 日勤: 09:00〜17:00、実働7時間、休憩1時間、色: 青系
  - 夜勤: 17:00〜09:00、実働14時間、休憩2時間、色: 紫系
- [ ] パターン一覧に2件表示、色・時間・工時が正しい
- [ ] パターン削除 → 確認ダイアログ → 削除される

---

## 3. メンバー管理 (Members)

- [ ] サイドバー「メンバー」→ メンバー一覧画面
- [ ] 「メンバー追加」→ 名前入力 + 対応パターン選択
- [ ] 5名作成（例: 田中、鈴木、佐藤、高橋、渡辺）、全員に日勤・夜勤を割当
- [ ] メンバー編集 → 名前変更が反映される
- [ ] メンバー削除 → 確認後に削除

### 3-1. Excel インポート
- [ ] 「テンプレートダウンロード」→ .xlsx がダウンロードされる
- [ ] テンプレートに5名分のデータを記入
- [ ] 「Excelインポート」→ プレビューモーダルで新規/更新件数が表示
- [ ] 「実行」→ メンバーが追加/更新される

---

## 4. グループ管理 (Groups)

- [ ] サイドバー「グループ」→ グループ一覧
- [ ] 「グループ追加」→ 名前 + メンバー選択（複数）
- [ ] グループ編集・削除が動作する

---

## 5. 個人制約 (Constraints)

- [ ] サイドバー「個人制約」→ 制約一覧
- [ ] メンバーを選択し、以下の制約を設定:
  - 期間出勤日数: 最小20日〜最大25日
  - 連続出勤上限: 5日
- [ ] 制約の編集・削除が動作する

---

## 6. スケジュール管理

### 6-1. スケジュール作成
- [ ] サイドバー「スケジュール」→ 一覧画面
- [ ] 「新規作成」→ 名前、開始日（2026-07-13）、日数（14日間）を入力
- [ ] 一覧に新しいスケジュールが表示される（ステータス: draft）

### 6-2. 毎日の必要人数
- [ ] スケジュールをクリック → 設定画面
- [ ] 「毎日の必要人数」セクション: 最小2人、最大4人で「一括設定」
- [ ] 「設定済み: 14日間」と表示される

### 6-3. 固定割当
- [ ] メンバー選択 → 日付選択 → タイプ「休み」→ 追加
- [ ] テーブルに固定割当が表示される
- [ ] 削除ボタンで削除できる

---

## 7. 休み希望受付フロー

### 7-1. 受付開始（管理者側）
- [ ] スケジュール設定画面「休み希望受付」セクション
- [ ] 「休み希望受付を開始」ボタンをクリック
- [ ] ステータスが「受付中」に変わる
- [ ] 全メンバーのリストが表示される（ステータス: 未提出）
- [ ] 各メンバーの「リンク」ボタン → 個人リンクモーダルが表示される
- [ ] リンクURLをコピーできる

### 7-2. 休み希望提出（個人ページ）
- [ ] コピーした個人リンクを **新しいブラウザタブ** で開く
- [ ] 「○○ さん」と名前が表示される
- [ ] スケジュールカードに「受付中」タグが表示される
- [ ] 「休み希望を選択」ボタン → カレンダーモーダルが開く
- [ ] 日付をタップ → 選択される（緑バッジ表示）
- [ ] 最大日数を超えて選択 → 警告メッセージ
- [ ] 「確定提出」ボタン → 成功メッセージ
- [ ] モーダルを閉じると「提出済み」タグと選択日が表示される

### 7-3. 管理者確認
- [ ] 管理者画面に戻り、リロード（またはF5）
- [ ] 提出済みメンバーのステータスが「提出済み」に変わっている
- [ ] 提出された希望日が表示されている
- [ ] 提出進捗（例: 1/5名）が更新されている

### 7-4. 受付締切
- [ ] 「受付を締め切る」ボタン → 確認ダイアログ → 実行
- [ ] 未提出メンバーが自動提出される
- [ ] スケジュールステータスが「draft」に戻る
- [ ] 「固定割当」テーブルに、提出された休み日が自動追加されている

---

## 8. 排班生成 + 結果表示

### 8-1. 排班生成
- [ ] スケジュール設定画面「スケジュール生成」ボタン
- [ ] ステータスが「生成中...」に変わる
- [ ] 数秒後、自動的に結果ページにリダイレクトされる

### 8-2. 結果ページ確認
- [ ] 統計カード: Health Score, ステータス(completed), 求解時間, 違反数
- [ ] スコア明細: 個人制約/グループ需要/毎日需要/均衡性のペナルティ表示
- [ ] 排班表: メンバー × 日付のグリッド
  - 出勤日: パターン名 + 色付きタグ
  - 休み日: 「REST」タグ（緑）
  - 固定割当の休みが正しく反映されている
- [ ] 違反レポート: 違反がある場合、種別/優先度/詳細が表示される

### 8-3. Excel エクスポート
- [ ] 「Excel出力」ボタン → .xlsx ファイルがダウンロードされる
- [ ] ファイルを開き、排班表・統計・違反レポートのシートが存在する
- [ ] 排班表のデータがブラウザ表示と一致する

### 8-4. 三方案比較
- [ ] スケジュール設定画面に戻り「三方案比較」ボタン
- [ ] 3つの方案（均衡/人力優先/個人優先）がカードで表示される
- [ ] 各方案のスコア、違反数、ペナルティバーが表示される

## 9. レスポンシブ対応（モバイル）

- [ ] ブラウザの開発者ツール（F12）でモバイルビュー（375px幅）に切替
- [ ] サイドバーが非表示、ハンバーガーメニューが表示される
- [ ] メニューをタップ → Drawer ナビゲーションが開く
- [ ] 各画面がモバイル幅に収まっている（横スクロール不要）
- [ ] 個人ページ（/personal/:token）がモバイルで見やすいレイアウト

---

## 10. マルチテナント分離

- [ ] **別のブラウザ（またはシークレットモード）** で新規組織を登録
- [ ] 組織Aで作成したメンバー/パターン/スケジュールが組織Bに表示されない
- [ ] 組織Aの個人リンクで、組織Bのスケジュールにはアクセスできない

---

## 11. エラーハンドリング

- [ ] 無効な個人リンク（/personal/invalid-token）→ エラーメッセージ表示
- [ ] ログイン失敗 → エラーメッセージ表示
- [ ] サーバー停止中にアクセス → 画面が白くならない（エラー表示）
